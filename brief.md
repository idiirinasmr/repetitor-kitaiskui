# Brief: Репетитор по китайскому — Telegram Mini App

## Общие требования

- Дизайн нативный Telegram (CSS-переменные `--tg-theme-*`)
- Крупные кнопки минимум 48px (touch target)
- Минимум текста, максимум визуала
- Русский язык интерфейса
- Иероглифы 28-40px, пиньинь 14px сверху
- Цветовая кодировка тонов: 1-красный, 2-оранжевый, 3-зелёный, 4-синий
- Загрузка < 1 сек, `expand()` + `ready()` при старте
- Бандл < 100KB (vanilla JS, без фреймворков)
- Сохранение прогресса в CloudStorage после каждого ответа
- HapticFeedback на каждое взаимодействие
- `showPopup()` вместо `alert()`, нативные диалоги
- Skeleton-загрузка вместо спиннеров
- Анимации переходов между экранами (CSS slide, 250ms)
- Микроанимации: bounce кнопок, пульсация, confetti
- `overscroll-behavior: none`, `-webkit-tap-highlight-color: transparent`
- `user-select: none` на интерактивных элементах
- Звук ТОЛЬКО по нажатию 🔊 (не автоплей)
- BackButton работает всегда и предсказуемо
- Шрифты: Inter (UI) + Noto Sans SC (иероглифы)

**5 типов упражнений:**

| Тип | ID | UI | Вопрос |
|-----|----|----|--------|
| Выбор перевода | `choice_translation` | 4 кнопки | Иероглиф → русский |
| Выбор иероглифа | `choice_hanzi` | 4 кнопки | Русский → иероглиф |
| Угадай тон | `tone_listen` | 4 кнопки с цветами | Послушай → тон |
| Сопоставление | `match_pairs` | Два столбца | Иероглиф ↔ перевод |

**Элементы:**
- Кнопка закрыть (✕) — выход из урока (с подтверждением)
- Прогресс-бар (6/10) — линейный, сколько осталось
- Вопрос (текст)
- Иероглиф крупно (32-40px) + пиньинь (14px)
- Кнопка 🔊 — прослушать (Web Speech API, zh-CN)
- 4 варианта ответа (кнопки 48px)
- **Кнопка "Не знаю"** — показывает правильный ответ без штрафа XP
- MainButton: "Проверить"

**Обратная связь — правильный ответ:**
```
┌─────────────────────┐
│ ✅ Правильно! +10XP │  ← зелёный фон
└─────────────────────┘
```
- Правильный вариант подсвечен зелёным
- Плашка снизу: "✅ Правильно! +10 XP"
- **XP-анимация**: "+10" всплывает и исчезает
- HapticFeedback: `notificationOccurred('success')`
- MainButton: "Продолжить"

**Обратная связь — неправильный ответ:**
```
┌─────────────────────┐
│ ❌ Неверно.         │  ← красный фон
│ 你好 (nǐ hǎo) =    │
│ Привет              │  ← объяснение!
└─────────────────────┘
```
- Неправильный — красным, правильный — зелёным
- Плашка снизу: объяснение с правильным ответом (иероглиф + пиньинь + перевод)
- HapticFeedback: `notificationOccurred('error')`
- MainButton: "Понятно"

| Функция | Почему не сейчас | Когда |

| **Словарь с поиском** | Можно спросить AI-чат | Phase 2 |
| **Интервальное повторение (SRS)** | Алгоритм SM-2 + бэкенд | Phase 2 |
| **Экзаменационный режим HSK** | Полный банк вопросов | Phase 3 |
| **Чтение историй** | Нужен контент | Phase 3 |
| **Офлайн-режим (Service Worker)** | Усложняет MVP | Phase 2 |
| **Streak Freeze** | Покупка за XP | Phase 2 |
| **Шаринг достижений** | Виральность | Phase 2 |
| **Лендинг (отдельная страница)** | Фокус на Mini App | Phase 2 |
| **Разделы кроме "Основы"** | Нужен контент | Phase 2 |

## Технический стек

| Компонент | Технология | Почему |
|-----------|------------|--------|
| Frontend | Чистый HTML + CSS + JS | < 100KB, быстрая загрузка |
| Telegram SDK | telegram-web-app.js (CDN) | Официальный SDK |
| Стили | CSS-переменные `--tg-theme-*` | Нативный вид |
| Данные | JSON-файлы (hsk1.json, lessons.json) | Не нужен бэкенд |
| Прогресс | Telegram CloudStorage | Встроенное хранилище |
| AI-чат | Мок (Phase 1) → Claude API (Phase 2) | — |
| Голос | Web Speech API (synthesis + recognition) | Бесплатно, встроено |
| Хостинг | GitHub Pages / Vercel | Бесплатно, HTTPS |
| Шрифт UI | Inter (Google Fonts) | Нейтральный, читаемый |
| Шрифт иероглифов | Noto Sans SC (Google Fonts) | Качественный |

## Порядок разработки

7. Дашборд (стрик, XP, текущий урок, разделы 2×2)

### Этап D — Упражнения
10. 5 типов упражнений + обратная связь (правильно/неправильно)

12. Итог урока (confetti, XP, список слов)
13. Прогресс: XP, стрик, достижения

### Этап E — AI-чат и голос
14. AI-чат (мок-ответы, chips, поле ввода)
