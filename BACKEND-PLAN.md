## Что строим

Telegram-бот с Mini App внутри. Бот делает три вещи:
1. **Принимает оплату** → открывает доступ к ступени
2. **Посредник для ДЗ** → ученик сдаёт, учитель проверяет
3. **Инструмент учителя** → видеть своих учеников, их прогресс, ДЗ

Mini App (фронтенд на Vercel) остаётся как есть. Бэкенд — это серверная логика бота.

## Роли пользователей

| Роль | Кто | Что может |
|------|-----|-----------|
| `student` | Обычный ученик | Проходить уроки, сдавать ДЗ, платить за доступ |
| `teacher` | Преподаватель (2-5 чел.) | Видеть своих учеников, проверять ДЗ, добавлять ссылки на уроки |
| `admin` | Владелец (вы) | Всё. Назначать учителей, управлять контентом, видеть всех |

## Где хранить данные

**Фаза 1 (до 50 учеников): Google Таблица**

Бесплатно. Вы видите всё в привычном интерфейсе. Бот пишет и читает через Google Sheets API.

```
Таблица 1: users       ← все пользователи и роли
Таблица 2: progress    ← прогресс каждого ученика
Таблица 3: access      ← кто к какой ступени имеет доступ
Таблица 4: payments    ← все платежи (вручную помечаете "paid")
Таблица 5: homework    ← сданные домашки и статус проверки
```

**Фаза 2 (50–500 учеников): PostgreSQL на Railway**

Бесплатный план до 500 МБ. Бот переключается с Google Sheets на SQL.
Миграция простая — структура данных та же, только переезжает в реальную БД.

## Как работает оплата (вручную → автомат)

### Фаза 1: Вручную (сейчас)

```
1. Ученик нажимает "Первая ступень" в боте
2. Бот отвечает:
   "Стоимость: 1990 ₽
    Переведите на карту: 1234 5678 9012 3456 (Ирина А.)
    После оплаты напишите /paid и пришлите скриншот"

3. Ученик переводит, присылает скриншот
4. Вы видите скриншот, проверяете
5. Пишете: /unlock @username step1
6. Бот автоматически открывает доступ и уведомляет ученика
```

### Фаза 2: Автомат (ЮKassa, позже)

```
1. Ученик нажимает "Купить Ступень 1"
2. Бот генерирует ссылку оплаты ЮKassa
3. Ученик платит картой
4. ЮKassa присылает webhook → бот получает сигнал
5. Бот АВТОМАТИЧЕСКИ записывает платёж и открывает доступ
6. Ученик получает уведомление: "Доступ открыт!"
```

**Что меняется при автоматизации:** только шаги 3-6. Структура данных та же. Добавляем ЮKassa и webhook — 1-2 дня работы.

## Как работают домашние задания

### Простые ДЗ (тесты) — автоматически

```
Ученик проходит тест в Mini App
→ App.js считает результат
→ Сохраняет в localStorage + отправляет в Google Sheets
→ В таблице: статус "auto_passed", score: 85
→ Учителю ничего делать не нужно
```

### Сложные ДЗ (голос, фото, письмо) — через бота

```
1. Ученик в Mini App нажимает "Сдать ДЗ"
2. Бот пишет: "Отправьте голосовое сообщение / фото / текст"
3. Ученик отправляет
4. Бот сохраняет file_id в Google Sheets, статус: pending
5. Учителю приходит уведомление:
   "📩 ДЗ от @anna_student
    Урок: Ступень 1 — Фонетика
    [голосовое 0:23]
    /review_123_good — Принять
    /review_123_redo — На доработку"

6. Учитель нажимает кнопку или пишет комментарий
7. Бот обновляет статус в таблице
8. Ученику приходит: "Ваше ДЗ проверено! Оценка: ✅"
```

Отдельный сервер с постоянным соединением (polling вместо webhook).
Подключается к PostgreSQL. Переезд — 1 рабочий день.

## Где хранится контент уроков

**Рекомендация: Google Drive + YouTube**

```
YouTube:
└── Плейлист "Ступень 1 — закрытый"
    ├── Урок 1: Тоны (видео, ссылка только по URL)
    ├── Урок 2: Согласные (видео)
    └── Урок 3: Гласные (видео)

Google Drive:
└── Папка "Ступень 1"
    ├── урок1_конспект.pdf
    ├── урок2_упражнения.pdf
    └── аудио_произношение.mp3
```

В базе данных (таблица step_content) хранятся только ссылки.
Вы добавляете контент через команду `/add_lesson` или прямо в Google Таблицу.

## Последовательность разработки

### Шаг 1 — Основа бота (1-2 дня)
- Команда `/start` регистрирует пользователя в Google Таблице
- Команда `/unlock` открывает доступ к ступени
- Mini App читает доступы и показывает/скрывает контент

### Шаг 2 — Ручная оплата (1 день)
- Команда `/buy step1` → инструкция по переводу
- Команда `/paid` → ученик присылает скриншот
- Вы проверяете → `/unlock @username step1`

### Шаг 3 — ДЗ через бота (2-3 дня)
- Ученик отправляет голосовое/фото боту
- Бот пересылает учителю с кнопками проверки
- Учитель нажимает → ученику приходит результат

### Шаг 4 — Прогресс учеников (1-2 дня)
- Mini App отправляет прогресс на бэкенд после каждого урока
- Команда `/students` для учителя
- Команда `/stats` для вас

### Шаг 5 — Ссылки на контент (1 день)
- Таблица step_content в Google Sheets
- Команда `/add_lesson` для добавления ссылок
- Mini App загружает ссылки динамически

### Шаг 6 — Автооплата ЮKassa (2-3 дня, когда будет ИП)
- Подключаем ЮKassa
- Webhook обрабатывает успешный платёж
- Бот автоматически открывает доступ
