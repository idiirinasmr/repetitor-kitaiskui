# Исследование: Каталог услуг репетитора как Telegram Mini App

> Репетитор по китайскому языку — Telegram Mini App с каталогом услуг, уроками, AI-чатом и системой прогресса.
> Дата исследования: март 2025

## 1. Анализ 5 Telegram Mini App в нише репетиторства и смежных

### 1.1 Durger King (@DurgerKingBot) — Эталонный каталог

**Что делает:** Официальное демо-приложение от Telegram — имитация заказа еды. Каталог товаров → корзина → оформление. Эталонный пример структуры «витрина + выбор».

**Сильные стороны:**
- Идеальный пример каталога с кнопкой «Добавить» — ровно то, что нужно для каталога услуг
- Чистый UI: карточки с фото, счётчик, MainButton для оформления
- Правильная интеграция Telegram SDK: MainButton, BackButton, тема
- Мгновенная загрузка, минимальный бандл
- Работает как нативное приложение

**Слабые стороны:**
- Слишком простой — нет профиля, истории, настроек
- Нет системы аккаунтов и персонализации
- Нет анимаций и микровзаимодействий
- Одноуровневая навигация (нет вложенных экранов)

**Что мы можем сделать лучше:**
- Вместо товаров — услуги репетитора с кнопкой «Выбрать»
- Добавить профиль, прогресс, многоуровневую навигацию
- Использовать HapticFeedback на каждое взаимодействие
- Добавить анимации переходов между экранами

### 1.2 Telegram Wallet — Полноценное приложение внутри Telegram

**Что делает:** Криптокошелёк и P2P-маркетплейс. Полноценное приложение: баланс → каталог → детали → оплата → история. Пример того, как Mini App может быть сложным и при этом удобным.

**Сильные стороны:**
- Tab Bar снизу — ощущается как нативное приложение
- Карточки с ценами, анимации переходов, pull-to-refresh
- Полноценная навигация со стеком экранов
- Профиль, история, настройки — всё как в обычном приложении
- Высочайшее качество UI/UX

**Слабые стороны:**
- Сложный для новичков — много функций сразу
- Перегруженный главный экран
- Нет онбординга для новых пользователей
- Долгая первоначальная загрузка

**Что мы можем сделать лучше:**
- Взять за образец Tab Bar и структуру навигации
- НЕ перегружать главный экран (один экран = одно действие)
- Добавить онбординг (цель + уровень), которого у Wallet нет
- Сделать загрузку моментальной через лёгкий vanilla JS

### 1.3 Салоны красоты (YCLIENTS/Dikidi через Telegram) — Запись на услугу

**Что делает:** Запись на услуги в салоны красоты через Mini App. Пошаговый wizard: выбор услуги → мастер → дата/время → подтверждение. Прямой аналог «каталога услуг репетитора».

**Сильные стороны:**
- **Пошаговый wizard** — идеальный паттерн для выбора услуги репетитора
- Карточки мастеров с фото и рейтингом → аналог «выберите раздел/уровень»
- Календарь с доступными слотами
- CTA-кнопка «Записаться» всегда на виду (MainButton)
- Простая и понятная структура

**Слабые стороны:**
- Медленная загрузка при большом количестве мастеров/услуг
- Нет геймификации — сухая функциональность
- Нет AI-помощника — только ручной выбор
- Интерфейс часто выглядит как мобильный сайт, а не как нативное приложение
- Нет отслеживания прогресса

**Что мы можем сделать лучше:**
- Использовать wizard-подход для первого знакомства (онбординг)
- Вместо «мастеров» — разделы: Основы / Разговорный / Грамматика / Деловой / Продвинутый
- Добавить AI-рекомендации: «Вам подойдёт раздел Основы»
- Геймификация: XP, стрики, достижения — чего нет ни в одном сервисе записи
- Skeleton-загрузка вместо спиннеров

### 1.4 GetGems (@getgems_bot) — Маркетплейс-витрина

**Что делает:** NFT-маркетплейс на TON-блокчейне внутри Telegram. Витрина коллекций → детали → покупка → профиль с коллекцией. Пример сложного каталога с поиском и фильтрами.

**Сильные стороны:**
- Полноценный каталог с поиском, фильтрами, сортировкой
- Trending/рекомендации на главной — персонализация
- Красивые карточки с превью
- История покупок и коллекция в профиле
- Нативный вид с плавными анимациями

**Слабые стороны:**
- Перегружен визуально — много информации на одном экране
- Специфичная аудитория (крипто)
- Нет обучающего элемента — предполагает, что пользователь всё знает
- Высокий порог входа

**Что мы можем сделать лучше:**
- Взять идею «коллекции» → наш аналог: «коллекция выученных иероглифов»
- Фильтры по разделам: HSK 1 / HSK 2 / HSK 3
- Персонализация: «Рекомендуем повторить», «Новые слова для тебя»
- Низкий порог входа: 30 секунд до первого упражнения

### 1.5 Telegram-боты для языков (@chinese_words_bot, @hsk_bot, @LearnChineseBot)

**Что делают:** Отправляют слова дня, проводят простые тесты через inline-кнопки в чате. Наши ближайшие конкуренты в Telegram.

**Сильные стороны:**
- Уже в Telegram — не нужно ничего скачивать
- Push-уведомления с новыми словами (используют бота)
- Простота: открыл → получил слово → ответил
- Бесплатные

**Слабые стороны:**
- **Примитивный интерфейс** — только текст и inline-кнопки в чате
- Нет красивого UI — всё через сообщения бота
- Нет интерактивности (нельзя слушать произношение, нет карточек)
- **Нет AI** — только заготовленные ответы по скрипту
- Нет системы прогресса, нет геймификации
- Нет голосового ввода/вывода
- Нет каталога услуг — это просто рассылка слов

**Что мы можем сделать лучше:**
- **Telegram Mini App = полноценный UI** внутри Telegram (карточки, анимации, Tab Bar)
- AI-наставник вместо скриптовых ответов
- Голосовой ввод/вывод через Web Speech API
- Геймификация как у Duolingo, но внутри Telegram
- Каталог услуг с кнопкой «Выбрать» — не просто бот, а приложение
- **Это наше ГЛАВНОЕ преимущество:** качество отдельного приложения + удобство Telegram

## 4. Пользовательский поток (User Flow)

### 4.1 Первый запуск

```
Пользователь видит бота → нажимает кнопку → открывается Mini App
  → Приветствие (3 сек, авто)
  → Выбор цели: путешествия / работа / экзамен / хобби
  → Выбор уровня: начинающий / средний / продвинутый
  → [Ежедневная цель: 5 / 10 / 15 / 20 мин] ← ДОБАВИТЬ
  → [Мини-урок: 3 лёгких вопроса — "aha moment"] ← ДОБАВИТЬ
  → Дашборд
```

**Критически важно:** Duolingo вставляет мини-урок (2-3 вопроса) ПРЯМО в онбординг. Пользователь получает «aha moment» — «Я уже что-то могу!» — до того, как увидит дашборд. Это увеличивает retention первого дня на 15-20%.

### 4.2 Каталог услуг репетитора

```
Дашборд → Нажимает раздел (напр. «Основы»)
  → Список вкладок-услуг:
    ┌─────────────────────────────────┐
    │ 📚 Урок 1: Числа 1-10          │
    │ 10 упражнений • 20 XP          │
    │                    [Выбрать ▶]  │
    ├─────────────────────────────────┤
    │ 📚 Урок 2: Приветствия          │
    │ 10 упражнений • 20 XP          │
    │                    [Выбрать ▶]  │
    ├─────────────────────────────────┤
    │ 🔒 Урок 3: Знакомство           │
    │ Пройдите урок 2 чтобы открыть  │
    │                    [Закрыт 🔒]  │
    └─────────────────────────────────┘
  → Нажимает «Выбрать» → Экран упражнения
  → 10 упражнений → Итог → Возврат в каталог
```

### 4.3 Ежедневная сессия

```
Push-напоминание от бота: "Не теряй стрик 🔥 — 5 мин до нового урока!"
  → Пользователь открывает Mini App
  → Дашборд: стрик 🔥12, кнопка «Продолжить урок»
  → Упражнения (7-10 шт, 3-5 мин)
  → Итог: "+20 XP! Стрик 🔥13!"
  → Возврат на дашборд
```

## 6. Геймификация

### 6.1 Механики

| Механика | Описание | Влияние на retention |
|----------|----------|---------------------|
| **Стрик** 🔥 | Дни подряд без пропуска | +40% DAU (данные Duolingo) |
| **XP** ⭐ | 10-20 очков за урок | Чувство прогресса |
| **Уровни** | Повышение при накоплении XP | Долгосрочная цель |
| **Достижения** 🏆 | За milestones (100 слов, 30 дней) | Коллекционирование |
| **Daily Quest** | Ежедневная цель (заработай 50 XP) | Причина вернуться |
| **Streak Freeze** | Заморозить стрик за XP | Снижает churn |

**Принцип:** Ранний успех. Первые 5 минут = ощущение прогресса. Пользователь должен выучить 3 слова и получить "+20 XP" до того, как увидит экран тарифов.

### 6.2 Что добавить к текущей реализации

1. **XP-анимация** — "+10 XP" всплывающий текст при каждом правильном ответе (сейчас показывается только в итоге)
2. **Confetti** — при завершении урока (CSS @keyframes)
3. **Звуковые эффекты** — короткий "ding" при правильном, "buzz" при неправильном (Web Audio API)
4. **Streak Freeze** — покупается за XP, защищает стрик на 1 день (Phase 2)
5. **Daily Goal Progress Ring** — кольцо "5 из 10 минут сегодня" на дашборде

## 7. Преимущества кастомной разработки над Lovable/no-code

### 7.1 Производительность

| Аспект | No-code (Lovable/Bubble) | Наш custom-build |
|--------|--------------------------|-------------------|
| **Размер бандла** | 500KB — 2MB | 50-100KB |
| **Время загрузки** | 2-5 сек | < 1 сек |
| **First Contentful Paint** | 1.5-3 сек | 0.3-0.5 сек |
| **Память** | 30-80MB | 5-15MB |
| **60fps анимации** | Тормозит на бюджетных Android | Плавно на любом устройстве |

**Почему это критично:** Mini App работает внутри WebView Telegram, который уже потребляет память. Каждый лишний MB — это лаг. Основная аудитория Telegram в СНГ — бюджетные Android-устройства.

### 7.2 Telegram-нативные фичи (НЕДОСТУПНЫ в no-code)

| Фича | Что даёт | Lovable? |
|------|----------|----------|
| `HapticFeedback` | Вибрация при ответах | Нет |
| `MainButton` | Нативная кнопка внизу | Частично |
| `BackButton` | Нативная навигация назад | Частично |
| `CloudStorage` | Хранение прогресса | Нет |
| `showPopup()` | Нативные алерты | Нет |
| `openInvoice()` | Оплата через Telegram Stars | Нет |
| CSS `var(--tg-theme-*)` | Авто-тема светлая/тёмная | Нет |
| `initDataUnsafe` | Имя, фото, язык пользователя | Нет |
| Web Speech API | Озвучка и распознавание речи | Нет |
| Canvas API | Рисование иероглифов | Нет |

### 7.3 UX-polish (мелочи, создающие качество)

- **Скелетон-загрузка** вместо спиннеров
- **Микроанимации кнопок** — `transform: scale(0.95)` при нажатии
- **Плавные переходы экранов** — CSS slide transitions (250ms)
- **overscroll-behavior: none** — убрать «резиновый» скролл
- **-webkit-tap-highlight-color: transparent** — убрать синие пятна при тапе
- **user-select: none** на интерактивных элементах
- **Оптимистичные обновления** — показать экран сразу, не дожидаясь загрузки

### 8.4 Ограничения и решения

| Ограничение | Решение |

### 11.1 UX-дизайнер

#### Типичные ошибки новичков (и как их избежать)

| # | Ошибка | Почему плохо | Как правильно |
|---|--------|-------------|---------------|
| 1 | **Каталог без иерархии** — все услуги в одном списке | Пользователь теряется в 20+ карточках, не понимает с чего начать | **Вкладки-разделы** вверху: Основы / Разговорный / Грамматика / Деловой / Продвинутый. Нажал вкладку → увидел только свои услуги |
| 2 | **Цена без контекста** — просто «690 ₽» | Не понятно за что, дорого или дёшево | Показать **что входит**: «10 уроков, 150 слов, AI-чат». Сравнение: «= 2 чашки кофе в месяц» |
| 3 | **Кнопка «Записаться» без предварительного просмотра** | Страх покупки «кота в мешке» | **Экран-превью услуги**: описание, что изучим (3-5 иероглифов), отзыв, длительность. И только потом «Записаться» |
| 4 | **Онбординг > 4 шагов** | Каждый лишний шаг теряет 20% пользователей | Максимум 3 экрана: приветствие → цель → уровень → сразу каталог |
| 5 | **Регистрация до ценности** | Пользователь ещё не понял зачем, а его просят данные | В Telegram **регистрация не нужна** — `initData` даёт ID, имя и фото автоматически |
| 6 | **Перегруженный главный экран** | 20 кнопок, 5 разделов, баннеры → паника → закрыл | **Один экран = одно действие.** Дашборд = «Продолжить урок» (одна главная кнопка) + краткая статистика |
| 7 | **Мелкие кнопки и иероглифы** | Промахиваешься пальцем, не видишь черты | Touch target минимум **48px**, иероглифы **28-40px**, пиньинь **14px** сверху |
| 8 | **Нет обратной связи** | Нажал → ничего → «Сломано?» → ушёл | Каждое действие = мгновенная реакция: цвет, анимация, `HapticFeedback` |
| 9 | **Одинаковый каталог для всех** | Новичок видит «HSK 3 Деловой» и пугается | **Персонализация**: после онбординга показать рекомендованный раздел первым. Остальные — ниже, с пометкой уровня |
| 10 | **Нет состояний у карточек услуг** | Все карточки одинаковые — не понятно что пройдено, что доступно | **3 состояния**: ✅ пройдено (серая, галочка), 🔵 доступно (яркая, «Выбрать»), 🔒 заблокировано (затемнённая, «Пройдите предыдущий») |

#### Обязательные элементы каталога услуг

**Карточка услуги (урока) — must-have поля:**
```
┌─────────────────────────────────────┐
│ 📚 Урок 3: Знакомство               │
│                                     │
│ Научитесь представляться по-китайски│

│                                     │
│ ⏱ 5 мин  •  📝 10 заданий  •  ⭐ 25 XP │
│                                     │
│           [ Выбрать ▶ ]             │
└─────────────────────────────────────┘
```

Элементы карточки:
1. **Название** — короткое, понятное («Знакомство», а не «Урок 3.2.1»)
2. **Описание** — одна строка, что научишься делать
3. **Превью контента** — 3-5 иероглифов, которые выучишь (визуальная ценность)
4. **Метрики** — время, количество заданий, XP-награда
5. **CTA-кнопка** — «Выбрать» / «Продолжить» / «Заблокировано»
6. **Статус** — пройден / доступен / заблокирован (визуально разные)

**Экран каталога — структура:**
```
┌─────────────────────────────────────┐
│ ← Назад          Каталог услуг      │
├─────────────────────────────────────┤
│ [Основы] [Разговор.] [Граммат.] ... │  ← горизонтальные chips
├─────────────────────────────────────┤
│                                     │
│  Карточка услуги 1 (✅ пройден)     │
│  Карточка услуги 2 (🔵 доступен)    │
│  Карточка услуги 3 (🔒 закрыт)      │
│  ...                                │
│                                     │
├─────────────────────────────────────┤
│  📚 Уроки    🤖 AI-чат    👤 Профиль │  ← Tab Bar
└─────────────────────────────────────┘
```

#### Что обязательно должно быть в приложении

1. **Прогресс-бар** в каждом уроке (вижу сколько осталось)
2. **BackButton** всегда работает и предсказуемо (Telegram native)
3. **Анимация правильного ответа** — зелёная вспышка + `HapticFeedback.success`
4. **Объяснение при ошибке** — не просто «Неправильно», а ПОЧЕМУ. Показать правильный ответ с пиньинь и переводом
5. **Кнопка «Не знаю»** в каждом упражнении — снижает фрустрацию. Показывает правильный ответ и идёт дальше (без штрафа XP)
6. **Кнопка 🔊** — прослушать произношение в любой момент
7. **Skeleton-загрузка** — серые плейсхолдеры вместо спиннеров
8. **Сохранение при каждом ответе** — закрыл/открыл = продолжил с того же места
9. **Плавные переходы** между экранами (CSS slide, 250ms)
10. **Микроанимации** — bounce кнопок, пульсация текущего урока, confetti при завершении

#### Что бесит пользователей (избегать категорически)

| Проблема | Почему бесит | Решение |
|----------|-------------|---------|
| Нельзя пропустить сложное | Застрял → злость → удалил | Кнопка «Не знаю» / «Пропустить» |
| Потеря прогресса при закрытии | Делал 5 минут, случайно закрыл — всё заново | `CloudStorage.setItem()` после каждого ответа |
| Навязчивые уведомления | «Ты не занимался 2 часа!!!» × 5 раз в день | Максимум 1 push в день, в настроенное время |

### 11.3 Telegram-разработчик

#### Что работает отлично в Telegram Mini App

**1. Авторизация из коробки (главное преимущество)**
- `initData` даёт user_id, имя, фото, язык — бесплатно и мгновенно
- Не нужна своя система логинов, email, пароли
- Пользователь «записывается» одним нажатием — мы уже знаем кто он
- Для каталога услуг это ИДЕАЛЬНО: нажал «Записаться» → мы знаем кто записался

**2. MainButton — нативная CTA**
- Кнопка «Записаться» / «Начать урок» / «Оплатить» внизу экрана
- Пользователи привыкли к ней — она всегда на виду, не скроллится
- Динамический текст: на каталоге «Записаться (690 ₽)», в уроке «Проверить»
- Состояния: loading (спиннер), disabled (серая), active (яркая)

**3. HapticFeedback — ощущение нативности**
```javascript
// Каталог — при нажатии на карточку услуги:
HapticFeedback.impactOccurred('light');

// Упражнение — правильный ответ:
HapticFeedback.notificationOccurred('success');

// Упражнение — неправильный ответ:
HapticFeedback.notificationOccurred('error');

// Tab Bar — переключение вкладки:
HapticFeedback.selectionChanged();

// Завершение урока / получение достижения:
HapticFeedback.notificationOccurred('success');
```

**4. Тема автоматически**
- CSS-переменные `--tg-theme-*` адаптируют всё к светлой/тёмной теме
- Не нужно делать переключатель — Telegram сам решает
- Карточки каталога, кнопки, текст — всё подстраивается

**5. CloudStorage — прогресс без сервера**
- До 1024 ключей, 4KB на значение — для прогресса и настроек хватит
- Привязано к аккаунту Telegram — работает на всех устройствах пользователя
- Сохранять: пройденные уроки, XP, стрик, ответы, настройки

**6. Telegram Stars — оплата без комиссии 30%**
- `openInvoice()` — нативная оплата внутри Mini App (Phase 2)
- Нет комиссии Apple/Google App Store (30%)
- Для каталога услуг: нажал «Записаться» → invoice → оплатил → доступ открыт

**7. Шаринг и виральность**
- Пользователь может поделиться ботом прямо в Telegram
- `switchInlineQuery()` — шаринг через inline-бота
- «Поделись достижением с другом» → виральное распространение

#### Ограничения и решения

| Ограничение | Влияние на каталог услуг | Решение |
|-------------|--------------------------|---------|
| **Нет нативных табов** | Нужен свой Tab Bar для разделов каталога | Свой Tab Bar в HTML/CSS (3 вкладки: Уроки, AI-чат, Профиль) + горизонтальные chips для разделов |
| **CloudStorage: 1024 ключа, 4KB** | Достаточно для прогресса, мало для данных уроков | Прогресс → CloudStorage, данные уроков → JSON-файлы на сервере |
| **Mini App закрывается при сворачивании** | Потеря состояния посреди урока | Сохранять после КАЖДОГО ответа: `CloudStorage.setItem('lesson_state', {...})` |
| **Push только через бота** | Нельзя слать уведомления из Mini App | Бот шлёт напоминания — **это даже лучше** (пользователь видит в чате) |
| **iOS: автоплей аудио заблокирован** | Нельзя автоматически произносить иероглифы | Воспроизводить ТОЛЬКО по нажатию кнопки 🔊 |
| **Web Speech API может не работать** | Голосовой ввод недоступен на некоторых устройствах | Fallback: текстовый ввод пиньинь + кнопки выбора тона |
| **Нет доступа к файлам устройства** | Не можем экспортировать словарь | Не критично — словарь внутри приложения |
| **Размер бандла** | Долгая загрузка = уход пользователя | Vanilla JS, JSON lazy-load, бандл < 100KB, кэш Service Worker |
| **Viewport на iOS** | Клавиатура перекрывает поля ввода | `visualViewport` API + скролл к активному полю |
| **Нет фонового режима** | Нельзя отслеживать время обучения в фоне | Засекать время на клиенте, сохранять при ответах |

#### Что лучше НЕ делать

| Антипаттерн | Почему плохо | Правильная альтернатива |
|-------------|-------------|------------------------|
| **React/Vue/Angular** | 500KB+ бандл, 2-5 сек загрузка в WebView | Vanilla JS или Preact (< 100KB, < 1 сек) |
| **`alert()`** | Выглядит как ошибка, ломает UX | `Telegram.WebApp.showPopup({title, message, buttons})` |
| **Блокировать BackButton** | Пользователь чувствует себя в ловушке | Всегда обрабатывать: `BackButton.onClick(goBack)` |
| **Свой header/шапка** | Telegram уже показывает имя бота вверху | Не дублировать — использовать `setHeaderColor()` |
| **Модальные окна поверх всего** | В WebView выглядят кривовато | Отдельный экран или `showPopup()` |
| **Auto-redirect после таймера** | «Записывайтесь, осталось 5 сек!» — манипуляция | Пользователь сам нажимает «Записаться» |
| **Infinite scroll в каталоге** | 5 разделов × 5 уроков = 25 карточек = нормально | Пагинация по разделам через chips |
| **Хранить пароли/токены в CloudStorage** | Небезопасно | Использовать `initData` для авторизации запросов к серверу |

#### Архитектура каталога услуг — техническая схема

```
Пользователь нажимает кнопку бота
  → Telegram открывает Mini App (index.html)
  → JS: Telegram.WebApp.expand() + ready()
  → JS: Проверяет CloudStorage → есть прогресс?
    ├─ НЕТ → Онбординг (цель → уровень → мини-урок → дашборд)
    └─ ДА  → Дашборд (продолжить с прошлого места)

Дашборд:
  → Кнопка «Продолжить урок» (главная CTA)
  → Разделы (chips: Основы / Разговорный / ...)
  → Нажал раздел → Каталог услуг (список уроков)

Каталог услуг:
  → Карточки уроков с кнопкой «Выбрать»
  → Нажал «Выбрать» → Экран превью урока
    → Описание + что изучим + длительность + цена/XP
    → MainButton: «Начать урок» (бесплатный) / «Записаться (690₽)» (платный)

Урок:
  → 10 упражнений → Итог → Возврат в каталог
  → CloudStorage.setItem() после каждого ответа

AI-чат:
  → Отдельная вкладка в Tab Bar
  → Диалог с наставником (Claude API)
  → Chips: «Объясни тоны», «Помоги с 你好»

Оплата (Phase 2):
  → Telegram.WebApp.openInvoice(url) → Telegram Stars
  → Callback: invoice_closed → разблокировать контент
```

## 14. Вывод

**Ниша свободна.** Нет ни одного качественного Telegram Mini App для изучения китайского с AI-наставником и каталогом услуг. Существующие TG-боты примитивны (только текст), а приложения (HelloChinese, Duolingo) требуют отдельной установки и стоят дороже.

**Наш продукт:** качество отдельного приложения + удобство Telegram + AI-наставник + голос + каталог услуг. Уникальное сочетание на рынке.

**Главное преимущество над Lovable/no-code:** нативная интеграция с Telegram SDK (HapticFeedback, MainButton, CloudStorage, Stars), мгновенная загрузка (< 1 сек vs 2-5 сек), и полный контроль над UX — всё то, что невозможно в no-code платформах.
